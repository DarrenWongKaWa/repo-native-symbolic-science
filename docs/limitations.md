# Limitations

## ENGINE_003 Caveats

1. **Mathematica Availability**: Optional backend; not required for core functionality
2. **Symbolic vs Numerical**: Numerical agreement does not establish symbolic equality
3. **Capability Safety**: Unsupported operations fail safely
4. **Cross-Engine Verification**: Documented usage boundary exists
5. **Alternate E2 Naming**: Alternate naming convention for E2 fixtures exists

## General Limitations

- The framework does not invent scientific assumptions
- The framework does not automatically authorize IBP
- The framework does not silently reorder scientific limits
- Automatic canonical promotion is forbidden
- Private scientific content is excluded from the public repository

## ORCH Controller Runtime Limitations

- The controller works through declared adapters and contracts; unsupported roles or validators fail closed
- Synthetic adapters are demonstrations, not claims of universal scientific automation
- Human authorization remains required at scientific gates
- The framework does not guarantee that every scientific task can be solved
- The controller does not verify scientific correctness — that is the verifier's role
- The controller does not edit symbolic output — it only validates structure and compliance
- An executor cannot verify its own output; claim authority is bounded per role
- The default verdict is FAIL; a PASS requires positive evidence (at least one check executed, all passed, no blocking findings)
